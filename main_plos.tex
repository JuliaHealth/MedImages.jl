\section*{Abstract}
\textbf{Background and Objectives:} Quantitative medical imaging requires software architectures that ensure high-throughput computational performance and strict mathematical coupling between voxel data and spatial metadata. Existing frameworks often face a "two-language" bottleneck, bridging high-level APIs with compiled backends, which complicates automatic differentiation (AD) and can lead to metadata drift. We introduce MedImages.jl, a native Julia ecosystem for differentiable 3D/4D medical image analysis.

\textbf{Methods:} MedImages.jl implements a typed data structure that intrinsically binds voxel data to spatial metadata (origin, spacing, direction). Core geometric transformations are implemented as native, differentiable Julia kernels compatible with GPU acceleration (KernelAbstractions.jl). We validated the framework through four critical axes: (1) biobank-scale throughput using HDF5, (2) GPU-accelerated spatial kernels, (3) end-to-end differentiability via SciML, and (4) clinical metadata consistency. The primary validation is a novel Universal Differential Equation (UDE) dosimetry pipeline.

\textbf{Results:} MedImages.jl achieved a 7.2$\times$ speedup over Python-based caching in a 100-subject benchmark. GPU spatial transformations reached up to 135$\times$ speedups over SimpleITK. Our UDE dosimetry model achieved a Pearson correlation of \textbf{0.957} against Monte Carlo simulations, outperforming established deep-learning baselines while running \textbf{10.5--13.6$\times$ faster} than equivalent Python-based SciML implementations.

\textbf{Conclusions:} By unifying spatial operations and metadata management within a single differentiable language, MedImages.jl enables high-fidelity clinical research without the ergonomic and performance overhead of hybrid frameworks.

\textbf{Keywords:} Medical Imaging, Julia, Scientific Machine Learning, Nuclear Medicine, Differentiable Programming, Spatial Metadata

\section{Introduction}
The development of MedImages.jl is driven by the need to address four converging challenges in contemporary medical imaging pipelines, particularly within Nuclear Medicine workflows that rely heavily on functional data (e.g., PET, SPECT) and anatomical references (CT, MRI).

\paragraph{Challenge 1: The Volume of Modern Biobank Datasets.}
A single whole-body PET/CT examination involves high-resolution volumes that must be co-registered and resampled. At biobank scale, preprocessing thousands of subjects becomes a dominant bottleneck. Traditional hybrid environments suffer from significant I/O overhead and metadata synchronization delays when preparing multimodal datasets.

\paragraph{Challenge 2: Execution Speed and the "Two-Language" Barrier.}
Researchers often prototype in high-level languages like Python but must rely on opaque C++ backends for execution. This "two-language problem" limits code inspectability and creates non-differentiable boundaries. Julia addresses this by compiling directly to efficient machine code via LLVM JIT, allowing spatial transformations to be executed rapidly across both CPU and GPU backends using native implementations.

\paragraph{Challenge 3: Differentiability and the Fragmented SciML Landscape.}
Emerging Scientific Machine Learning (SciML) paradigms require pipelines where every step is differentiable. While Python offers powerful differentiable libraries, they often function as "walled gardens." Julia allows native code to seamlessly interoperate through multiple dispatch, enabling end-to-end differentiability across complex differential equation solvers and GPU kernels.

\paragraph{Challenge 4: Metadata Management and Physical Interpretation.}
Unlike standard computer vision, medical images possess a strict "physical interpretation." A medical image is a grid of points occupying a physical region defined by origin, spacing, and direction. The transition to raw arrays for deep learning often strips this spatial context, risking "metadata drift"—a dangerous misalignment between voxel data and coordinates.

\subsection*{Architectural Foundations: The Physical Coordinate System}
To address Challenge 4, software must rigorously maintain the mapping between discrete voxel indices $\mathbf{v}$ and physical patient space $\mathbf{p}$ via a $4 \times 4$ homogeneous affine transformation matrix $M$:
\begin{equation}
\mathbf{p} = M \mathbf{v}, \quad M = 
\begin{bmatrix} 
R S & T \\
\mathbf{0} & 1
\end{bmatrix}
\end{equation}
where $R$ is rotation, $S$ is scaling (spacing), and $T$ is translation (origin). MedImages.jl ensures that any spatial operation $\mathcal{T}$ applied to the voxel grid is coupled with an update $M_{new} = M_{old} \times M_{\mathcal{T}}$, preventing desynchronization.

\section{Methods}

\subsection*{Software Architecture: Zero-Cost Metadata Abstractions}
MedImages.jl implements a BIDS-inspired metadata paradigm where the \texttt{MedImage} struct enforces metadata integrity through Julia's type system. We use ITK (C++) only for I/O robustness, while all transformations are native Julia. This ensures (1) \textbf{Differentiability} via Zygote.jl/Enzyme.jl and (2) \textbf{Composability} with Lux.jl and DifferentialEquations.jl. To support high-throughput multimodal studies, \texttt{BatchedMedImage} stores 4D voxel data with vectorized metadata, facilitating efficient GPU operations.

\subsubsection*{Importance of SUV and Metadata Integrity}
Standardized Uptake Value (SUV) calculation is a physics problem depending entirely on clinical metadata:
\begin{equation}
SUV = \frac{C(measured) \cdot e^{\lambda \cdot \Delta t}}{Injected Dose / Patient Weight}
\end{equation}
Losing metadata like "Acquisition Time" or "Patient Weight" renders the image clinically useless. MedImages.jl intrinsically protects these fields.

\subsection*{Experimental Setup: Evaluating the Four Challenges}

\subsubsection*{Challenge 1 and 2: Volume and Speed}
We developed a fused affine preprocessing pipeline that collapses orientation normalization, resampling, and spatial centering into a single matrix. We benchmarked a 100-case preprocessing pipeline on the autoPET dataset against MONAI's \texttt{PersistentDataset}.

\subsubsection*{Challenge 3: Differentiability via UDE Dosimetry}
To validate differentiability, we implemented:
\begin{enumerate}
    \item \textbf{Learned Inverse Rotation:} A CNN learns to invert unknown 3D rotations via differentiable interpolation.
    \item \textbf{Organ-Specific Affine Registration:} A CNN predicts per-organ affine parameters, differentiated via \texttt{Enzyme.autodiff}.
    \item \textbf{UDE Dosimetry:} A 4-state system modeling $^{177}$Lu-PSMA dosimetry over 300 hours, discovering non-local transport residuals $\mathcal{N}_\theta$.
\end{enumerate}

\section{Discussion and Conclusions}
MedImages.jl provides a performant, differentiable ecosystem for medical imaging. By bridging the divide between acquisition physics and inference logic, it empowers clinicians to utilize high-fidelity SciML models that are order-of-magnitude faster than hybrid frameworks. The success of the UDE framework ($r=0.957$) proves that native high-performance ecosystems are the future of quantitative theranostics.
