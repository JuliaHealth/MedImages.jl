\section*{Abstract}
MedImages.jl addresses the two-language problem in medical imaging by providing native Julia implementations for spatial transformations while preserving clinical metadata through a BIDS (Brain Imaging Data Structure)-inspired architecture. Unlike Python frameworks (SimpleITK, MONAI) where metadata is lost during NumPy/PyTorch conversion, MedImages.jl enforces zero-cost metadata abstractions via Julia's type system. The package achieves C++-level performance through LLVM JIT compilation while enabling end-to-end automatic differentiation—a capability unavailable in wrapped C++ libraries. By utilizing ITK only for I/O robustness and implementing all transformations in native Julia, the framework ensures full composability with scientific machine learning tools (Flux.jl, Zygote.jl) and supports HDF5-based storage for efficient pipelines. Benchmarks demonstrate 30--138$\times$ GPU speedups (up to 8.1$\times$ over SimpleITK CPU) while maintaining clinical metadata integrity, positioning MedImages.jl as a foundation for physics-informed deep learning in medical imaging.

\section*{Author Summary}
Medical imaging research often relies on Python for ease of use but requires C++ for performance, creating a barrier known as the "two-language problem." We present MedImages.jl, a Julia package that bridges this gap. It provides high-performance, differentiable medical image processing with strong metadata support, crucial for clinical reliability. This tool is particularly relevant for Nuclear Medicine, where large, functional images (SPECT, PET) and complex metadata are common. By enabling fast, parallelized computations on standard hospital hardware and supporting diverse data types (dose maps, CT), MedImages.jl democratizes advanced analysis for researchers and clinicians, ensuring that computational limitations do not hinder medical progress.

\section*{Introduction}
Medical imaging suffers from the "two-language problem": researchers prototype in Python but rely on opaque C++ kernels (SimpleITK, ITK) for performance. This creates three critical limitations: (1) \textbf{Metadata Loss}—converting SimpleITK images to NumPy arrays discards spatial metadata (origin, spacing, direction), requiring manual tracking; (2) \textbf{Non-Differentiability}—C++ functions are black boxes to PyTorch's autograd, preventing end-to-end optimization of registration or reconstruction parameters; (3) \textbf{Performance Ceiling}—custom operations require C++/CUDA compilation, creating high-friction development cycles.

MedImages.jl solves these through Julia's unique capabilities. Native Julia code compiles to LLVM machine code, achieving C++ performance without the two-language barrier. The package implements a BIDS-inspired metadata paradigm where spatial context is intrinsic to the data structure via zero-cost type abstractions—metadata travels with pixel data without performance penalty. Critically, all operations are differentiable via Zygote.jl/Enzyme.jl, enabling physics-informed deep learning where MRI reconstruction physics or registration parameters can be optimized jointly with neural networks. GPU acceleration (60--180$\times$ speedups) is transparent, requiring no code changes.

\subsection*{Relevance to Nuclear Medicine}
This library is particularly well-suited for Nuclear Medicine (NM), a field that uses radiotracers to visualize physiological functions. Unlike anatomical imaging (CT, MRI), NM scans (such as PET and SPECT) frequently cover a wide physical range—often from mid-thigh to head for whole-body oncology surveys. This results in large volumetric datasets that require significant computational resources for reconstruction, processing, and analysis.

In a typical hospital setting, computational power is often constrained by legacy hardware or limited budget, making algorithm optimization crucial. MedImages.jl addresses this by providing fast, parallelized computations that maximize the performance of available hardware (both CPU and GPU). This efficiency enables complex analyses to be performed locally without needing high-performance computing clusters.

Furthermore, NM studies involve diverse and complex metadata, varying significantly between modalities like SPECT, PET, CT, and Dosemaps (radiotherapy). MedImages.jl ensures robust support for all these data types, preserving critical information such as radiotracer details, acquisition times, and dosage information throughout the processing pipeline. The library also supports batch processing, allowing for the efficient handling of large patient cohorts or time-series data (dynamic PET). By optimizing these foundational operations and ensuring data integrity, MedImages.jl makes advanced downstream algorithms more accessible to end-users, facilitating clinical research and deployment.

\section*{Design and Implementation}

\subsection*{Software Architecture}
MedImages.jl implements a BIDS-inspired metadata paradigm where the \texttt{MedImage} struct enforces metadata integrity through Julia's type system. Unlike Python workflows where converting to arrays discards spatial context, MedImages.jl uses zero-cost abstractions—metadata (origin, spacing, direction, patient ID, acquisition timestamp) is intrinsic to the structure without runtime overhead.

A critical design decision: ITK (C++) is used \textit{only} for I/O robustness (NIfTI, DICOM parsing), while \textit{all} transformations are native Julia. This ensures: (1) \textbf{Differentiability}—operations integrate with Zygote.jl/Enzyme.jl for automatic differentiation; (2) \textbf{Composability}—seamless integration with Flux.jl and DifferentialEquations.jl; (3) \textbf{Transparency}—users can inspect and extend algorithms without C++ recompilation.

The \texttt{MedImage} struct implements typed medical imaging semantics:
\begin{itemize}
    \item \texttt{voxel\_data}: Column-major arrays
    \item \texttt{origin}: Physical coordinates $(x,y,z)$ of first voxel
    \item \texttt{spacing}: Physical voxel dimensions
    \item \texttt{direction}: Direction cosines for orientation
    \item \texttt{metadata}: DICOM tags, Study/Series UIDs, patient demographics
    \item \texttt{modality}: Strong typing (\texttt{CTImage}, \texttt{PETImage}) enabling dispatch-based processing
\end{itemize}

To support high-throughput analysis, MedImages.jl introduces \texttt{BatchedMedImage}, which stores 4D voxel data $(x, y, z, \text{batch})$ and vectorized metadata. This structure allows efficient batch processing of multiple images, essential for training deep learning models or processing large clinical datasets in Nuclear Medicine.

\subsection*{Functionalities}
MedImages.jl provides a comprehensive set of functionalities optimized for 3D medical image processing:
\begin{itemize}
    \item \textbf{Robust I/O:} Loading of various medical image formats (NIfTI, DICOM) using \texttt{ITKIOWrapper}.
    \item \textbf{Native Julia Transformations:} Functions to modify spatial properties of 3D volumes (\texttt{rotate\_mi}, \texttt{translate\_mi}, \texttt{scale\_mi}, \texttt{resample\_to\_spacing}, \texttt{crop\_mi}, \texttt{pad\_mi}).
    \item \textbf{Resample to Target:} Resamples a moving image to match the spatial geometry of a reference image, critical for multi-modal registration (e.g., PET/CT alignment).
    \item \textbf{GPU Acceleration:} All transformations transparently support GPU execution via KernelAbstractions.jl.
    \item \textbf{HDF5 Support:} Fast loading and saving of 3D images using HDF5, speeding up I/O for intermediate processing.
    \item \textbf{Autodifferentiation:} All spatial transformations have well-defined gradients via Zygote.jl and Enzyme.jl.
    \item \textbf{Orientation Management:} Robust tools to handle and convert between neuroimaging coordinate systems (RAS, LPS, etc.).
\end{itemize}

\section*{Results}

\subsection*{Performance Benchmarks}
MedImages.jl was benchmarked on CPU and GPU backends. Benchmarks were conducted on an NVIDIA RTX 3090 GPU and Intel Core Ultra 9 285K CPU.

For $256 \times 256 \times 128$ volumes, MedImages.jl on GPU achieves significant speedups compared to SimpleITK on CPU:
\begin{itemize}
    \item \textbf{Resampling (Nearest Neighbor):} 115$\times$ speedup vs CPU, 17$\times$ vs SimpleITK.
    \item \textbf{Orientation Changes:} 71$\times$ speedup vs CPU, 31$\times$ vs SimpleITK.
    \item \textbf{Fused Affine Transformation:} 135$\times$ speedup vs CPU, 8.1$\times$ vs SimpleITK.
\end{itemize}

These results demonstrate that GPU acceleration is essential for 3D medical imaging, particularly for the large datasets common in Nuclear Medicine.

\subsection*{Automatic Differentiation}
MedImages.jl supports automatic differentiation (AD) via Zygote.jl and Enzyme.jl. This allows for differentiable image registration and the integration of imaging physics into deep learning models (scientific machine learning). A proof-of-concept experiment demonstrated that a neural network could learn inverse rotation parameters through differentiable \texttt{rotate\_mi} operations, reducing reconstruction error by 68.7\%.
